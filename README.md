# KawaCode - ランディングページ

## 📋 プロジェクト概要

KawaCode（CodeAwareness）の公式ランディングページです。AI時代のソフトウェア開発における「ユニバーサル・プログラミング」と「AIガバナンス」のコンセプトを訴求するために、GitHub風の洗練されたダークモードデザインで構築されています。

### 🎯 主要コンセプト

1. **意図（Intent）の管理** - AIが生成したコードの「なぜ」を永続化
2. **ユニバーサル・プログラミング** - 母国語で思考し、英語で保存
3. **AIガバナンス** - 監査可能な透明性とセキュリティ
4. **リアルタイム協調** - 未来の競合を予知する特許技術

## 🚀 ローカル開発

このプロジェクトは静的サイトのため、Webサーバーで直接開けます。

### シンプルな方法

```bash
# index.htmlをブラウザで直接開く
open index.html
```

### 推奨方法（ローカルサーバー）

```bash

npx http-server -p 8000
```

その後、ブラウザで `http://localhost:8000` を開きます。

## 📊 ROIシミュレーターの計算ロジック

### 入力パラメータ

- **従業員数（エンジニア）**: 1名以上
- **平均年収**: 100万円以上
- **現状の手戻り率**: 0-100%

### 計算式

```javascript
年間コスト = 従業員数 × 平均年収 × 10,000
現状の損失 = 年間コスト × (手戻り率 / 100)
改善後の手戻り率 = 手戻り率 / 2  // 50%削減を想定
年間削減コスト = 現状の損失 - (年間コスト × 改善後の手戻り率 / 100)
削減時間 = 総労働時間 × (手戻り率削減分 / 100)
実質増員効果 = 削減時間 / 2000時間
```

## 📧 お問い合わせフォームの設定

お問い合わせページ（`contact.html`）は、静的サイトでメール送信を実現するために、複数の方法をサポートしています。

### 設定方法

**推奨：Web3Forms（無料）**

1. [Web3Forms](https://web3forms.com/) でアカウントを作成
2. アクセスキーを取得
3. `contact.html` の以下の行を更新：

```html
<input type="hidden" name="access_key" value="YOUR_WEB3FORMS_ACCESS_KEY">
```

**代替：Formspree（無料）**

1. [Formspree](https://formspree.io/) でアカウントを作成
2. フォームIDを取得
3. `js/contact.js` の以下の行を更新：

```javascript
const response = await fetch('https://formspree.io/f/YOUR_FORMSPREE_ID', {
```

**フォールバック：Mailto**

設定なしの場合、ユーザーのメールクライアントが自動的に開き、`okamoto@kawacode.jp` へのメールが作成されます。

### フォーム機能

- **バリデーション**: 必須項目のチェックとメールアドレス形式の検証
- **エラーハンドリング**: 送信失敗時の適切なエラーメッセージ
- **ローディング状態**: 送信中のボタン無効化とローディングアニメーション
- **成功メッセージ**: 送信完了後のフィードバック表示
